01 — FOUNDATION

Strings in Java

In Java, a String is not a primitive type — it is an object of the java.lang.String class. Strings are immutable, meaning once a String object is created, its content can never be changed. Every operation that appears to modify a string actually creates a brand new String object in memory. This is fundamental to understanding Java's memory model.

String Pool & Immutability

Java maintains a special memory area called the String Constant Pool (inside the Heap). When you write String s = "hello", Java checks the pool first. If "hello" already exists there, it reuses that reference. This saves memory but means two variables can point to the same object.

String Pool vs new String()
Java
String s1 = "hello";       // Goes into String pool
String s2 = "hello";       // Reuses same pool reference
String s3 = new String("hello"); // Always creates new Heap object

System.out.println(s1 == s2);   // true  (same pool reference)
System.out.println(s1 == s3);   // false (different objects)
System.out.println(s1.equals(s3)); // true  (same content)

// Immutability demo
String original = "TCS";
original.concat(" Prime");   // returns NEW string, original unchanged
System.out.println(original);   // Still prints: TCS
String result = original.concat(" Prime");  // Capture the new string
💡

TCS Interview Tip: Always use .equals() to compare String content, never ==. The == operator checks object reference (memory address), not the actual characters inside.

String vs StringBuilder vs StringBuffer

This is a very common TCS interview question. Since String is immutable, repeatedly concatenating strings in a loop creates many objects. StringBuilder and StringBuffer are mutable alternatives.

Important String Methods

Returns number of characters. String index is 0-based.

Returns char at index i. Throws StringIndexOutOfBoundsException if i is invalid.

substring(s, e)

Returns substring from index s (inclusive) to e (exclusive).

indexOf(str)

Returns first occurrence of str, or -1 if not found.

contains(str)

Returns boolean — whether str is present inside.

Returns new String with all occurrences replaced.

Splits string by delimiter, returns String array.

Removes leading and trailing whitespace.

toUpperCase() / toLowerCase()

Case conversion. Returns new String object.

compareTo()

Lexicographic comparison. Returns 0 if equal, negative/positive otherwise.

Arrays

An array is a fixed-size, contiguous block of memory that stores elements of the same data type. Arrays in Java are objects — they live on the Heap. The array variable itself stores a reference (pointer) to this heap memory. Array size is fixed at creation and cannot be changed.

Array Declaration & Initialization

Arrays — all forms
Java
// 1D Arrays
int[] arr1 = new int[5];           // All zeros by default
int[] arr2 = {10, 20, 30, 40};       // Direct initialization
String[] names = {"Alice", "Bob"};

System.out.println(arr2.length);    // 4 (property, not method!)
System.out.println(arr2[0]);         // 10 (0-indexed)

// 2D Arrays (Matrix)
int[][] matrix = new int[3][3];
int[][] grid = {{1,2},{3,4},{5,6}};

// Jagged arrays (rows of different length)
int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[4];
jagged[2] = new int[1];

Iterating Arrays

Iteration patterns
Java
int[] nums = {5, 3, 8, 1, 9};

// Classic for loop
for(int i = 0; i < nums.length; i++) System.out.println(nums[i]);

// Enhanced for-each (read-only)
for(int n : nums) System.out.println(n);

// Java Arrays utility class
Arrays.sort(nums);                    // In-place sort, O(n log n)
System.out.println(Arrays.toString(nums)); // [1, 3, 5, 8, 9]
int idx = Arrays.binarySearch(nums, 5); // Binary search (array must be sorted)
int[] copy = Arrays.copyOf(nums, nums.length);
Arrays.fill(nums, 0);                  // Fill all with 0

// 2D matrix traversal
for(int i = 0; i < grid.length; i++)
    for(int j = 0; j < grid[i].length; j++)
        System.out.print(grid[i][j] + " ");

03 — OOP

Object-Oriented Programming

OOP is a programming paradigm that organizes software around objects — bundles of data (attributes) and behavior (methods). Java is a purely object-oriented language (except for primitives). TCS Prime heavily tests OOP concepts — expect at least 5–8 questions from this area alone.

🔒

Encapsulation

Wrapping data & methods together. Hiding internal state using private fields and public getters/setters.

🧬

Inheritance

Child class inherits properties and behavior from parent class. Promotes code reuse with extends.

🎭

Polymorphism

One interface, many implementations. Method Overloading (compile-time) and Overriding (runtime).

🎨

Abstraction

Hiding implementation details, showing only essential features. Achieved via abstract classes and interfaces.

04 — OOP

Classes & Objects

A class is a blueprint or template. An object is an instance — a real, living copy of that blueprint created in memory. Think of class as a cookie cutter and objects as the actual cookies.

Class anatomy
Java
public class Employee {
    // Fields (instance variables) — belong to each object
    private int id;
    private String name;
    private double salary;

    // Constructor — called when object is created
    public Employee(int id, String name, double salary) {
        this.id = id;
        this.name = name;
        this.salary = salary;
    }

    // Methods — behavior of the object
    public void work() {
        System.out.println(name + " is working");
    }

    // Getters and Setters
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    // Override toString for readable output
    @Override
    public String toString() {
        return "Employee{" + id + ", " + name + ", $" + salary + "}";
    }
}

// Creating objects
Employee e1 = new Employee(101, "Rahul", 75000);
Employee e2 = new Employee(102, "Priya", 85000);
e1.work();
System.out.println(e2);

The this keyword

Inside an instance method or constructor, this refers to the current object. It is used to distinguish between instance variables and constructor/method parameters when they share the same name. It can also be used to call one constructor from another (constructor chaining) with this(args).

05 — OOP

Inheritance

Inheritance is the mechanism by which one class (child/subclass) acquires the properties and behaviors of another class (parent/superclass) using the extends keyword. Java supports single and multilevel inheritance for classes, but NOT multiple inheritance (a class can't extend two classes — this is solved by interfaces).

Inheritance hierarchy
Java
// Parent class
class Animal {
    String name;
    void eat() { System.out.println(name + " is eating"); }
    void sleep() { System.out.println(name + " is sleeping"); }
}

// Child class — inherits eat() and sleep()
class Dog extends Animal {
    void bark() { System.out.println("Woof!"); }

    @Override
    void eat() {                          // Overrides parent behavior
        System.out.println(name + " eats dog food");
    }
}

// Multilevel: GuideDog → Dog → Animal
class GuideDog extends Dog {
    void guide() { System.out.println("Guiding owner safely"); }
}

// Using super keyword
class Cat extends Animal {
    @Override
    void eat() {
        super.eat();  // calls Animal's eat() first
        System.out.println("...then licks paws");
    }
}

Dog d = new Dog();
d.name = "Bruno";
d.eat();   // Dog's overridden eat()
d.sleep(); // Inherited from Animal

The super keyword

super refers to the parent class. It has two main uses: super.method() to call a parent class method that was overridden, and super(args) in a constructor to call the parent class constructor (must be the first statement).

06 — OOP

Polymorphism

Polymorphism means "many forms." The same method name can behave differently depending on context or the object calling it. Java has two types: Compile-time (Static) Polymorphism via method overloading, and Runtime (Dynamic) Polymorphism via method overriding.

Method Overloading (Compile-time)

Same class, same method name, different parameters (different number, type, or order). Java decides which method to call at compile time by looking at the argument types.

Method Overloading
Java
class Calculator {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; }
}
// Compiler knows which to call based on args
// Note: Return type ALONE cannot differentiate overloads

Method Overriding (Runtime)

Child class redefines a method inherited from parent. The method to call is decided at runtime based on the actual object type, even if the reference type is parent. This is the heart of runtime polymorphism.

Runtime Polymorphism — Key Demo
Java
class Shape {
    void draw() { System.out.println("Drawing a Shape"); }
}
class Circle extends Shape {
    @Override
    void draw() { System.out.println("Drawing a Circle ⬤"); }
}
class Rectangle extends Shape {
    @Override
    void draw() { System.out.println("Drawing a Rectangle ▬"); }
}

// UPCASTING — parent reference, child object
Shape s1 = new Circle();     // Valid! Circle IS-A Shape
Shape s2 = new Rectangle(); // Valid!
s1.draw();  // "Drawing a Circle"  — decided at RUNTIME
s2.draw();  // "Drawing a Rectangle" — based on actual object

// This is the POWER of polymorphism — write generic code
Shape[] shapes = { new Circle(), new Rectangle(), new Circle() };
for(Shape sh : shapes) sh.draw(); // Each calls correct draw()

07 — OOP

Encapsulation

Encapsulation means bundling data (fields) and methods that operate on that data into a single unit (class), and restricting direct access to the data. It's implemented by making fields private and providing public getter/setter methods. This is data hiding in action.

Encapsulation with validation
Java
public class BankAccount {
    private String accountNumber;
    private double balance;     // Cannot be accessed directly from outside!

    public BankAccount(String accNo, double initialBalance) {
        this.accountNumber = accNo;
        this.balance = initialBalance;
    }

    public double getBalance() { return balance; } // Read-only access

    public void deposit(double amount) {
        if(amount > 0) balance += amount;  // Validation inside setter
        else System.out.println("Invalid deposit amount");
    }

    public void withdraw(double amount) {
        if(amount > 0 && amount <= balance) balance -= amount;
        else System.out.println("Insufficient funds or invalid amount");
    }
}
// account.balance = -99999; // COMPILE ERROR — private!
// account.deposit(-500);     // Runtime validation catches this

Why Encapsulation?

1. Data Protection — fields cannot be set to invalid states directly. 2. Flexibility — internal implementation can change without affecting outside code. 3. Maintainability — changes are localized. 4. Reusability — well-encapsulated classes are easier to reuse.

08 — OOP

Abstraction

Abstraction is about hiding complexity and showing only what's necessary. You drive a car without knowing how the engine works — that's abstraction. In Java, it's achieved via abstract classes and interfaces. An abstract class can have both abstract (no body) and concrete (with body) methods.

Abstract classes
Java
abstract class Vehicle {
    String brand;

    // Abstract method — no body, MUST be overridden by child
    abstract void start();

    // Concrete method — available to all children
    void stop() { System.out.println(brand + " stopped"); }
}

class Car extends Vehicle {
    Car(String brand) { this.brand = brand; }

    @Override
    void start() { System.out.println(brand + ": Turn key → Engine starts"); }
}

class ElectricCar extends Vehicle {
    ElectricCar(String brand) { this.brand = brand; }

    @Override
    void start() { System.out.println(brand + ": Press button → Silent start"); }
}

// Vehicle v = new Vehicle(); // ERROR! Cannot instantiate abstract class
Vehicle v = new Car("Toyota"); // OK — polymorphism
v.start();   // Toyota: Turn key → Engine starts

Abstract Class vs Interface — Key Difference

Abstract class can have state (instance variables with values), constructors, and both abstract and concrete methods. Interfaces (before Java 8) could only have abstract methods and constants. Java 8+ interfaces can have default and static methods. A class can implement multiple interfaces but extend only one abstract class.

09 — OOP

Interfaces

An interface is a contract — it defines what a class must do, not how. It's the mechanism for achieving multiple inheritance and full abstraction in Java. All fields in an interface are implicitly public static final. All methods are implicitly public abstract (before Java 8).

Interfaces & Multiple Implementation
Java
interface Flyable {
    void fly();  // implicitly public abstract

    // Java 8+ default method — provides implementation
    default void land() { System.out.println("Landing safely..."); }

    // Java 8+ static method
    static void info() { System.out.println("This entity can fly"); }
}

interface Swimmable {
    void swim();
}

// Multiple interfaces — solving the diamond problem
class Duck implements Flyable, Swimmable {
    @Override
    public void fly() { System.out.println("Duck flaps wings"); }
    @Override
    public void swim() { System.out.println("Duck paddles"); }
}

// Interface as type reference (polymorphism)
Flyable f = new Duck();
f.fly();
f.land();  // Uses default method
Flyable.info(); // Static method call on interface itself

// Functional Interface (Java 8) — exactly 1 abstract method
interface Greeting {
    void greet(String name);
}
// Lambda — implements functional interface inline
Greeting g = name -> System.out.println("Hello, " + name);
g.greet("TCS");

10 — OOP

Constructors

A constructor is a special method that is automatically called when an object is created using new. It has the same name as the class and no return type (not even void). Its purpose is to initialize the object's state.

Constructor types
Java
class Student {
    String name; int age; double gpa;

    // 1. Default Constructor (no-arg)
    Student() {
        name = "Unknown"; age = 0; gpa = 0.0;
    }

    // 2. Parameterized Constructor
    Student(String name, int age, double gpa) {
        this.name = name; this.age = age; this.gpa = gpa;
    }

    // 3. Constructor Chaining with this()
    Student(String name) {
        this(name, 18, 0.0); // Calls parameterized constructor — must be FIRST line
    }

    // 4. Copy Constructor
    Student(Student other) {
        this.name = other.name;
        this.age  = other.age;
        this.gpa  = other.gpa;
    }
}

Student s1 = new Student();
Student s2 = new Student("Rahul", 20, 8.5);
Student s3 = new Student(s2);  // Deep copy

Important: If you define any constructor, Java no longer provides the default one!

Java automatically provides a no-arg default constructor only if you define zero constructors in your class. The moment you write any constructor (even just a parameterized one), the compiler removes the auto-generated default. Always explicitly define the no-arg constructor if you need it alongside a parameterized one.

Static & Final Keywords

The static Keyword

static means the member belongs to the class itself, not to any individual object. Static variables are shared across all instances. Static methods can be called without creating an object.

static keyword
Java
class Counter {
    static int count = 0;  // Shared across all objects
    int id;

    Counter() { count++; id = count; }

    static void showCount() {  // Static method — no 'this' reference
        System.out.println("Total objects: " + count);
        // System.out.println(id); // ERROR — cannot access instance var
    }

    // Static block — runs ONCE when class is loaded
    static {
        System.out.println("Counter class loaded!");
    }
}

Counter c1 = new Counter();
Counter c2 = new Counter();
Counter.showCount(); // Total objects: 2

The final Keyword

final means cannot be changed/overridden/extended. Applied to variables (constant), methods (can't override), and classes (can't extend).

final keyword
Java
final double PI = 3.14159;  // Constant — cannot reassign
// PI = 3.0; // COMPILE ERROR

class MathUtils {
    final void compute() { }  // Cannot be overridden in subclasses
}

final class Immutable { }   // Cannot be extended (e.g., String is final!)
// class MyString extends Immutable { } // COMPILE ERROR

// final reference — reference can't change but object can!
final StringBuilder sb = new StringBuilder("Hello");
sb.append(" World"); // OK — modifying the object
// sb = new StringBuilder(); // ERROR — can't reassign reference

Exception handling patterns
Java
// Basic try-catch-finally
try {
    int[] arr = new int[5];

14 — COLLECTIONS

HashMap

HashMap stores data as key-value pairs. It uses a hash function to compute an index (bucket) from the key's hashCode, giving average O(1) time for get and put. Keys must be unique — duplicate keys overwrite values. One null key is allowed; multiple null values are allowed. HashMap is not thread-safe.

How HashMap Works Internally

Internally, HashMap uses an array of LinkedLists (or Trees in Java 8+ when bucket size exceeds 8). When you call put(key, value): 1) hashCode() of key is computed, 2) it's mapped to a bucket index via hash % capacity, 3) if there's a collision, the entry is chained in the bucket's LinkedList.

HashMap operations
Java
Map<String, Integer> map = new HashMap<>();

// CRUD operations
map.put("Alice", 90);
map.put("Bob", 85);
map.put("Alice", 95);  // Overwrites Alice's value

System.out.println(map.get("Bob"));           // 85
System.out.println(map.getOrDefault("Zara", 0)); // 0 (key not found)
System.out.println(map.containsKey("Alice"));   // true
System.out.println(map.containsValue(85));      // true
map.remove("Bob");
System.out.println(map.size());                // 1

// Iteration patterns
for(Map.Entry<String,Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " → " + entry.getValue());
}
map.forEach((k, v) -> System.out.println(k + ": " + v)); // Lambda style

// Frequency count (classic TCS problem)
String[] words = {"apple", "banana", "apple", "cherry", "banana", "apple"};
Map<String, Integer> freq = new HashMap<>();
for(String w : words)
    freq.put(w, freq.getOrDefault(w, 0) + 1);
// {apple=3, banana=2, cherry=1}

Stack & Queue

Stack — LIFO (Last In, First Out)

Think of a stack of plates — you can only add or remove from the top. Used in function call management, undo operations, expression evaluation, and balanced parentheses problems.

Stack operations
Java
import java.util.Stack;

Stack<Integer> stack = new Stack<>();
stack.push(10);  // Add to top
stack.push(20);
stack.push(30);

System.out.println(stack.peek()); // 30 — look top without removing
System.out.println(stack.pop());  // 30 — remove top
System.out.println(stack.pop());  // 20
System.out.println(stack.isEmpty()); // false

// Preferred: Use Deque as Stack (faster, modern)
Deque<Integer> dStack = new ArrayDeque<>();
dStack.push(1);   dStack.push(2);
dStack.peek();    dStack.pop();

// Classic: Balanced parentheses
boolean isBalanced(String s) {
    Stack<Character> st = new Stack<>();
    for(char c : s.toCharArray()) {
        if(c == '(' || c == '{' || c == '[') st.push(c);
        else if(st.isEmpty()) return false;
        else if(c == ')' && st.pop() != '(') return false;
        else if(c == '}' && st.pop() != '{') return false;
        else if(c == ']' && st.pop() != '[') return false;
    }
    return st.isEmpty();
}

Queue — FIFO (First In, First Out)

Think of a ticket queue — first person in line is served first. Used in BFS algorithms, scheduling, print queues, and message processing.

Queue & PriorityQueue
Java
import java.util.*;

// LinkedList as Queue
Queue<String> queue = new LinkedList<>();
queue.offer("First");   // Add to rear (prefer offer over add)
queue.offer("Second");
queue.offer("Third");

System.out.println(queue.peek());  // "First" — view front
System.out.println(queue.poll());  // "First" — remove front
System.out.println(queue.poll());  // "Second"

// PriorityQueue — elements ordered by natural order (min-heap)
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(30); pq.offer(10); pq.offer(20);
System.out.println(pq.poll()); // 10 — smallest first

// Max-heap PriorityQueue
PriorityQueue<Integer> maxPQ = new PriorityQueue(Collections.reverseOrder());
maxPQ.offer(30); maxPQ.offer(10); maxPQ.offer(20);
System.out.println(maxPQ.poll()); // 30 — largest first

// Deque — double-ended queue
Deque<Integer> deque = new ArrayDeque<>();
deque.addFirst(1);  deque.addLast(2);
deque.peekFirst(); deque.peekLast();
deque.pollFirst(); deque.pollLast();

JDBC — Full CRUD example
Java
import java.sql.*;

public class JDBCDemo {
    // Step 1: Define connection details
    static final String URL = "jdbc:mysql://localhost:3306/mydb";
    static final String USER = "root";
    static final String PASS = "password";

    public static void main(String[] args) {
        // Step 2: Load Driver (optional in JDBC 4.0+, auto-loaded)
        // Class.forName("com.mysql.cj.jdbc.Driver");

        // Step 3: Establish Connection
        try(Connection conn = DriverManager.getConnection(URL, USER, PASS)) {

            // Step 4: Create Statement
            Statement stmt = conn.createStatement();

            // CREATE
            stmt.executeUpdate("INSERT INTO employees VALUES(1, 'Rahul', 75000)");

            // READ — executeQuery returns ResultSet
            ResultSet rs = stmt.executeQuery("SELECT * FROM employees");
            while(rs.next()) {
                System.out.println(rs.getInt("id") + " | " +
                                   rs.getString("name") + " | " +
                                   rs.getDouble("salary"));
            }

            // PreparedStatement — PREFERRED (prevents SQL injection)
            String sql = "INSERT INTO employees(id, name, salary) VALUES(?, ?, ?)";
            PreparedStatement ps = conn.prepareStatement(sql);
            ps.setInt(1, 2);
            ps.setString(2, "Priya");
            ps.setDouble(3, 85000);
            ps.executeUpdate();

            // Transaction Management
            conn.setAutoCommit(false);  // Disable auto-commit
            try {
                stmt.executeUpdate("UPDATE accounts SET bal=bal-500 WHERE id=1");
                stmt.executeUpdate("UPDATE accounts SET bal=bal+500 WHERE id=2");
                conn.commit();  // Both succeed — commit
            } catch(Exception e) {
                conn.rollback(); // Either fails — rollback both
            }

        } catch(SQLException e) {
            e.printStackTrace();
        } // Connection auto-closed (try-with-resources)
    }
}

  • What is SQL Injection? — Malicious SQL via user input. PreparedStatement prevents it by treating inputs as data, not code
  • What is a transaction? — A sequence of operations treated as a single unit; either ALL succeed (commit) or ALL fail (rollback)
  • ACID properties — Atomicity, Consistency, Isolation, Durability

Hibernate & ORM

ORM (Object-Relational Mapping) is a technique to map Java objects to database tables automatically, eliminating verbose JDBC code. Hibernate is the most popular ORM framework for Java. Instead of writing SQL, you work with Java objects and Hibernate generates the SQL behind the scenes.

The Core Idea

With plain JDBC, you write SQL + Java + ResultSet parsing manually. With Hibernate, you annotate your Java class and Hibernate handles the SQL, connection management, and result mapping. One class = One table. One object = One row.

Hibernate Entity & Annotations

Hibernate Entity class
Java
import jakarta.persistence.*;

@Entity                    // Marks this class as a DB table entity
@Table(name = "employees") // Maps to 'employees' table
public class Employee {

    @Id                    // Primary key
    @GeneratedValue(strategy = GenerationType.IDENTITY) // Auto-increment
    private int id;

    @Column(name = "emp_name", nullable = false, length = 100)
    private String name;

    @Column(name = "salary")
    private double salary;

    @ManyToOne             // Many employees → One department
    @JoinColumn(name = "dept_id")
    private Department department;

    // Getters, Setters, No-arg constructor required by Hibernate
}

Session & CRUD with Hibernate

Hibernate SessionFactory & CRUD
Java
// SessionFactory — heavyweight, created once at startup
SessionFactory sf = new Configuration()
    .configure("hibernate.cfg.xml")
    .buildSessionFactory();

// Session — lightweight, one per request/transaction
try(Session session = sf.openSession()) {
    Transaction tx = session.beginTransaction();

    // CREATE — persist to DB
    Employee emp = new Employee();
    emp.setName("Rahul");
    emp.setSalary(75000);
    session.save(emp);  // or session.persist(emp)

    // READ by primary key
    Employee e = session.get(Employee.class, 1);

    // UPDATE — just modify, Hibernate detects dirty state
    e.setSalary(80000);  // Auto-updated on commit if session is open

    // DELETE
    session.delete(e);

    // HQL — Hibernate Query Language (object-oriented SQL)
    List<Employee> list = session
        .createQuery("FROM Employee WHERE salary > :sal", Employee.class)
        .setParameter("sal", 70000)
        .getResultList();

    tx.commit();
}

Key Hibernate Concepts for TCS

Lazy vs Eager Loading

Lazy: related data loaded only when accessed (saves queries). Eager: loads everything upfront. Default is Lazy for collections.

Caching

L1 Cache = Session cache (always on). L2 Cache = SessionFactory-level (optional, shared). Reduces DB round trips.

HQL

Hibernate Query Language — like SQL but uses class names and field names instead of table/column names. Database-independent.

Cascade Types

ALL, PERSIST, MERGE, REMOVE, REFRESH. Controls what happens to related entities when you perform operations on the parent.

Full layered architecture — Employee example
Java
// 1. ENTITY (Model Layer) — maps to DB table
@Entity
public class Employee {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;
    private double salary;
    // getters, setters, constructors...
}

// 2. REPOSITORY (Data Access Layer) — extends JpaRepository
@Repository
public interface EmployeeRepository extends JpaRepository<Employee, Long> {
    // Spring Data JPA provides: findAll(), findById(), save(), deleteById()
    // Custom query by method name:
    List<Employee> findBySalaryGreaterThan(double salary);
    Optional<Employee> findByEmail(String email);
}

// 3. SERVICE (Business Logic Layer)
@Service
public class EmployeeService {
    @Autowired  // Dependency Injection
    private EmployeeRepository repo;

    public List<Employee> getAllEmployees() { return repo.findAll(); }

    public Employee getById(Long id) {
        return repo.findById(id)
            .orElseThrow(() -> new RuntimeException("Employee not found"));
    }

    public Employee createEmployee(Employee emp) { return repo.save(emp); }

    public void deleteEmployee(Long id) { repo.deleteById(id); }
}

// 4. CONTROLLER (Presentation Layer)
@RestController
@RequestMapping("/api/employees")
public class EmployeeController {
    @Autowired
    private EmployeeService service;

    @GetMapping
    public List<Employee> getAll() { return service.getAllEmployees(); }

    @GetMapping("/{id}")
    public Employee getOne(@PathVariable Long id) { return service.getById(id); }

    @PostMapping
    public Employee create(@RequestBody Employee emp) { return service.createEmployee(emp); }

    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) { service.deleteEmployee(id); }
}

Dependency Injection (DI): @Autowired tells Spring to inject the required bean automatically. Instead of you calling new EmployeeRepository(), Spring creates and injects it. This is the Inversion of Control (IoC) principle — the framework controls object creation, not your code.

REST APIs — What is an API?

An API (Application Programming Interface) is a contract that allows two software systems to communicate. Think of it as a waiter in a restaurant — you (client) give your order (request) to the waiter (API), who goes to the kitchen (server) and brings back your food (response). A REST API follows the REST architectural style using HTTP methods.

What is REST?

REST (Representational State Transfer) is an architectural style for designing APIs. It uses standard HTTP methods, is stateless (server stores no session info), and typically exchanges data in JSON format. Resources are identified by URLs.

@RestController
@RequestMapping("/api/employees")
public class EmployeeController {
    @Autowired private EmployeeService service;

    @GetMapping
    public ResponseEntity<List<Employee>> getAll() {
        return ResponseEntity.ok(service.getAllEmployees()); // 200
    }

    @GetMapping("/{id}")
    public ResponseEntity<Employee> getById(@PathVariable Long id) {
        try {
            return ResponseEntity.ok(service.getById(id));  // 200
        } catch(RuntimeException e) {
            return ResponseEntity.notFound().build();     // 404
        }
    }

    @PostMapping
    public ResponseEntity<Employee> create(@RequestBody Employee emp) {
        Employee saved = service.createEmployee(emp);
        return ResponseEntity.status(201).body(saved);  // 201 Created
    }

    @PutMapping("/{id}")
    public ResponseEntity<Employee> update(
            @PathVariable Long id, @RequestBody Employee emp) {
        emp.setId(id);
        return ResponseEntity.ok(service.createEmployee(emp)); // 200
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        service.deleteEmployee(id);
        return ResponseEntity.noContent().build();  // 204
    }
}
